             PHASE 1: BSM LITERATURE SURVEY
═══════════════════════════════════════════════════════════════════════════════

PAPER FINDER (Primary Method — Use First)
─────────────────────────────────────────────────────────────────────────────

    python .claude/skills/paper-finder/scripts/find_papers.py "your BSM topic"

Example:
    python .claude/skills/paper-finder/scripts/find_papers.py "scalar singlet dark matter Higgs portal HL-LHC invisible"

Modes:
    --mode fast       Quick search
    --mode diligent   Thorough (recommended for BSM — literature is dense)

STEP 1: Identify BSM Class and Search Keywords
─────────────────────────────────────────────────────────────────────────────

From the hypothesis, extract:
- SM problem being targeted (hierarchy, DM, neutrino mass, flavor, g−2, etc.)
- BSM class (supersymmetry, scalar extension, extra gauge, leptoquark, axion,
  sterile neutrino, composite, EFT, etc.)
- Target final state or observable (mono-X, displaced vertex, LFV, neutrino
  oscillation, relic density, direct-detection cross-section, etc.)
- Target facility (HL-LHC, Belle II, DUNE, LZ, IceCube, CMB-S4, FCC, etc.)

Build a keyword list combining these axes. Include synonyms: "SUSY" ↔ "supersymmetry",
"DM" ↔ "dark matter", "NP" ↔ "new physics", "BSM" ↔ "beyond the standard model",
"ALP" ↔ "axion-like particle", "HNL" ↔ "heavy neutral lepton" ↔ "sterile neutrino".

STEP 2: Search Sources
─────────────────────────────────────────────────────────────────────────────

✓ arXiv (https://arxiv.org) — hep-ph (primary), hep-ex (for experiment results),
  hep-th (for UV completions), astro-ph.CO (for cosmo/DM), nucl-ex/nucl-th (for
  nuclear matrix elements in DM/ν searches)

✓ INSPIRE-HEP (https://inspirehep.net) — citation-network search, author disambiguation,
  best tool for finding "what cites this paper" and "what does this paper cite".
  Essential for placing a proposal in its literature context.

✓ Semantic Scholar — good for broader coverage

✓ Google Scholar — fallback

✓ Snowmass 2021 reports (arXiv listings under various EF/TF/CF/NF/RF/IF frontiers)
✓ European Strategy Update documents
✓ P5 2023 Report (US) — priorities for near-future facilities

STEP 3: Download and Organize
─────────────────────────────────────────────────────────────────────────────

For each relevant paper:
1. Download PDF to papers/
   - arXiv: https://arxiv.org/pdf/{arxiv_id}.pdf
2. Name descriptively: papers/{first_author}_{year}_{short_topic}.pdf
3. Log in papers/README.md

TARGETS (aim for each category):
- 3–5 foundational papers for the BSM class (the canonical references)
- 3–5 recent (last 3 years) phenomenology papers with similar spirit
- 2–3 experimental papers providing the strongest exclusion limits relevant to the model
- 1–2 Snowmass / community reports for projected facility sensitivities
- User-specified papers (highest priority — never skip)

STEP 4: Extract Key Information
─────────────────────────────────────────────────────────────────────────────

For each paper, note:

✓ BSM model (Lagrangian or particle-content summary)
✓ Free parameters and their ranges
✓ Constraints applied by the authors and their sources
✓ Tools used (MadGraph, FeynRules, Pythia, Delphes, micrOMEGAs, SModelS, etc.)
✓ Numerical exclusion / discovery results (luminosity, significance, mass reach)
✓ Code / UFO model availability
✓ Gaps the authors themselves acknowledge

DEEP READING STRATEGY:
- User-specified papers and direct prior-art papers: read every chunk
- Papers that provide exclusion limits we plan to recast: read the analysis section carefully, note cuts and signal-region definitions
- Other papers: skim abstract + conclusion + key figures

        PHASE 2: CONSTRAINT DATA AND HEPDATA COLLECTION
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Identify Constraint Needs
─────────────────────────────────────────────────────────────────────────────

Based on the proposed model, which constraints matter? Typically a subset of:

- LHC direct searches at 13 TeV, Run 2 (139 fb⁻¹)
- LHCb / Belle II flavor results
- EWPO (Z-pole precision, S/T/U parameters, W mass)
- Lepton universality tests
- Cosmology (Planck Ω_DM h², N_eff, BBN)
- Direct DM detection (LZ, XENONnT, PandaX-4T, DarkSide-50, SuperCDMS)
- Indirect DM detection (Fermi-LAT, HESS, AMS-02, IceCube)
- Charged-lepton flavor violation (μ→eγ, μ→e conversion, τ LFV)
- EDMs (electron, neutron)
- (g−2)_μ Fermilab

STEP 2: Pull Data from Authoritative Sources
─────────────────────────────────────────────────────────────────────────────

✓ HEPData (https://www.hepdata.net) — digitized exclusion curves, likelihoods,
  correlation matrices. Many ATLAS/CMS/LHCb analyses deposit their full data here.
  API: `pip install hepdata-cli` or direct REST endpoints.
  Download: JSON or YAML tables per analysis.

✓ PDG (https://pdg.lbl.gov) — world-averages for SM parameters and standard limits.
  Download the current Review of Particle Physics summary tables (PDF or data files).

✓ ATLAS / CMS / LHCb public results pages — exclusion plots, sometimes only available
  as figures. Use WebPlotDigitizer if needed to extract curves.

✓ SModelS database — pre-digitized LHC search topologies ready for recasting.
  Clone: git clone https://github.com/SModelS/smodels.git code/smodels
  And: git clone https://github.com/SModelS/smodels-database.git code/smodels-database

✓ Planck 2018 parameters — Ω_DM h² = 0.120 ± 0.001, N_eff = 2.99 ± 0.17, etc.
  Source: https://wiki.cosmos.esa.int/planck-legacy-archive (or arXiv:1807.06209)

✓ LZ, XENONnT, PandaX-4T — latest SI and SD DM–nucleon cross-section upper limits.
  Typically published in PRL/PRD with accompanying HEPData entries or plots to digitize.

✓ Fermi-LAT dwarf-spheroidal limits — annihilation cross-section vs. mass.
  https://www-glast.stanford.edu/pub_data/

✓ Muon g−2 Fermilab — current Δa_μ world-average.

✓ MEG II, Mu2e, COMET — current and projected limits on charged LFV.

STEP 3: Digitize and Store
─────────────────────────────────────────────────────────────────────────────

Save constraints in structured, loadable formats:

- datasets/constraints/{source}_{observable}.{json,csv,yaml}
- Document units, central value, uncertainty, date/publication, luminosity/exposure
- Where a likelihood is available (HEPData "Full Likelihoods"), prefer it over a
  single-point limit — it lets us compute p-values properly
- Flag which constraints are most likely to exclude the proposed model; these should
  be checked first in the experiment runner phase

STEP 4: Gitignore Large Data
─────────────────────────────────────────────────────────────────────────────

```bash
cat > datasets/.gitignore << 'EOF'
*
!.gitignore
!README.md
!**/README.md
!**/samples/
!**/download.sh
!**/download_instructions.txt
!constraints/**
EOF
```

Small digitized constraint files are fine to commit; large raw experimental
ntuples are not.
