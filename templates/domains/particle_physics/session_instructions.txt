You MUST use an isolated environment for this project. DO NOT use the

REQUIRED STEPS (in order):

1. Check if a virtual environment already exists (the resource finder may have created one):

   If .venv/ directory exists:
   - Skip to step 3 (activate it)
   - The pyproject.toml should already exist

   If .venv/ does NOT exist:
   - Create a fresh virtual environment using uv:
     uv venv

2. Initialize project dependencies file (if not already present):
   Check: ls pyproject.toml 2>/dev/null || echo "NEED_PYPROJECT"

   If pyproject.toml does NOT exist:

   cat > pyproject.toml << 'EOF'
   [project]
   name = "research-workspace"
   version = "0.1.0"
   description = "Particle physics (BSM) research workspace"
   requires-python = ">=3.10"
   dependencies = []

   [build-system]
   requires = ["hatchling"]
   build-backend = "hatchling.build"
   EOF

   CRITICAL: This ensures uv won't search parent directories for pyproject.toml
   and contaminate the neurico environment!

3. Activate the environment:
   source .venv/bin/activate

4. For package installations, use this priority order:

   FIRST CHOICE - uv add (manages pyproject.toml automatically):
   uv add <package-name>

   Examples useful for BSM phenomenology:
   - uv add numpy scipy matplotlib pandas
   - uv add flavio           # Flavor observables (B-physics, LFV, etc.)
   - uv add iminuit          # Maximum-likelihood fits
   - uv add pyslha           # SLHA spectrum file I/O
   - uv add pypdf            # PDF chunker
   - uv add sympy            # Symbolic Lagrangian manipulation if needed

   SECOND CHOICE - uv pip install <package-name>
   LAST RESORT - pip install <package-name>

   NEVER use conda or conda install!

   Heavy HEP tools (MadGraph, Pythia, Delphes, FeynRules, SModelS, micrOMEGAs,
   CheckMATE, MadAnalysis5) are NOT Python packages — they are built from source
   or extracted from tarballs. The resource finder should have cloned them under
   code/. Build / install them there as needed; do NOT try to pip-install them.

5. Dependency Management:
   - 'uv add' updates pyproject.toml automatically
   - Verify with: cat pyproject.toml
   - If you used pip: pip freeze > requirements.txt
   - Document non-Python tool versions explicitly in REPORT.md

WHY: Using an isolated environment ensures:
- No pollution of the neurico environment
- Reproducibility of the BSM analysis
- Clean tracking of all dependencies
